# Como contribuir

Obrigado pelo interesse em contribuir.

## Formas de contribuição

Você pode contribuir com:

- correções de documentação;
- melhoria de clareza do código;
- ajustes no ambiente Webots;
- correções de bugs do controlador;
- novos mundos de teste e cenários experimentais.

## Boas práticas

- preserve a proposta didática do exemplo;
- documente mudanças de lógica de controle;
- descreva claramente qualquer alteração em `μ`, `λ`, `Gc`, `Gct` ou nas rotinas R1...R13;
- evite mudanças grandes sem contextualização.

## Fluxo sugerido

1. Faça um fork do repositório.
2. Crie uma branch descritiva.
3. Realize as alterações.
4. Atualize a documentação quando necessário.
5. Abra um Pull Request com descrição objetiva.

## Reporte de bugs

Ao abrir uma issue, inclua:

- versão do Webots;
- sistema operacional;
- mundo executado;
- trecho do log de debug;
- comportamento esperado e comportamento observado.
