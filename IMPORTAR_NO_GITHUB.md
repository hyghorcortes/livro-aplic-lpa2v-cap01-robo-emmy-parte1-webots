# Como publicar este projeto no GitHub

## Nome sugerido do repositório

```text
livro-lpa2v-cap01-robo-emmy-parte1-webots
```

## Descrição curta sugerida

```text
Exemplo em Webots do Robô Emmy - Parte 1 do livro de LPA2v e LF.
```

## Topics sugeridos

```text
lpa2v, paraconsistent-logic, webots, robotics, mobile-robot, python, para-analyzer, emmy-robot, book-example
```

## Opção A — Criar no GitHub e subir pela interface web

1. Crie um novo repositório vazio no GitHub.
2. Use o nome sugerido acima.
3. Não adicione README, `.gitignore` ou licença na tela inicial, porque estes arquivos já estão neste pacote.
4. Após criar o repositório vazio, envie o conteúdo desta pasta.

## Opção B — Criar pelo Git local

Dentro desta pasta, execute:

```bash
git init
git branch -M main
git add .
git commit -m "Initial commit: Robô Emmy - Parte 1 (Webots)"
git remote add origin https://github.com/SEU-USUARIO/livro-lpa2v-cap01-robo-emmy-parte1-webots.git
git push -u origin main
```

## Sobre o que já está pronto neste pacote

Este pacote já inclui:

- documentação principal do projeto;
- licença;
- metadados de citação;
- arquivos de apoio à colaboração;
- estrutura de repositório organizada.

## Ajustes finais recomendados no GitHub

Depois de publicar, ajuste no campo **About**:

- descrição curta;
- URL do projeto ou do livro, se houver;
- topics sugeridos.

Também é recomendável fixar uma imagem social do repositório usando uma captura do mundo do Webots ou uma imagem do robô.
